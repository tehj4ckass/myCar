## 2024-05-18 - Parameterized Queries for Environment Variables in SQLite
**Vulnerability:** SQL injection risk through f-string concatenation when inserting environment variables (like `VIN`) into SQLite queries. While environment variables might seem safe, using f-strings bypasses SQLite's built-in escaping, allowing potentially malicious values (e.g., if an attacker manages to set `VIN` to `'; DROP TABLE messages; --`) to alter the SQL syntax and cause a severe security breach.
**Learning:** Even internal environment variables that aren't strictly direct user input need to be treated as untrusted data when building SQL queries. The codebase used to use Python `f"{VIN}"` directly inside queries.
**Prevention:** Always use parameterized queries combined with SQLite string concatenation. For wildcard LIKE queries or topic generation, write `LIKE '%' || ? || '/charging/power'` and pass `VIN` in the parameters tuple. Do not rely on f-strings for passing variables into SQL.
