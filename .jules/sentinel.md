## 2024-11-20 - SQL Injection Vulnerability via Environment Variable Interpolation
**Vulnerability:** SQL queries in dashboard/pages/laden.py and dashboard/pages/trips.py were using f-strings to directly interpolate the 'VIN' environment variable into the query string, which could lead to SQL injection if the environment variable is attacker-controlled or maliciously formatted.
**Learning:** Even inputs sourced from the environment (like 'VIN') must be treated as untrusted and properly parameterized when used in SQL queries. In SQLite, this often means using string concatenation (e.g. `LIKE '%' || ? || '/charging/power'`) instead of string interpolation in the Python code.
**Prevention:** Always use parameterized queries (e.g. `?` placeholders) and avoid using Python f-strings or string concatenation for any part of the query structure that incorporates variable data. Use SQLite string concatenation operator (`||`) inside the query when combining parameterized inputs with string literals like in `LIKE` clauses.
