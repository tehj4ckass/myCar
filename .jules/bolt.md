## 2023-10-24 - Streamlit UI interactions trigger massive DB queries without caching
**Learning:** Streamlit reruns the entire page script on almost any UI interaction (e.g. changing tabs, clicking buttons). When combined with `streamlit_autorefresh`, functions like `latest()`, `detect_trips()`, and `history()` run multiple times and trigger a large number of expensive SQLite queries, causing a major performance bottleneck in this specific architecture.
**Action:** Always wrap data-fetching and expensive transformation functions (like those querying SQLite and building Pandas DataFrames) with `@st.cache_data(ttl=2)` for a "debounce" effect. This ensures the app feels real-time but limits DB hits to at most once every 2 seconds per function signature, mitigating N+1 query storms during rapid UI interactions.
